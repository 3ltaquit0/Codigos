Tipos de Excepciones en Programación

Definición

Las excepciones son eventos inesperados que ocurren durante la ejecución de un programa y que interrumpen su flujo normal. Dependiendo de cómo se manejen estas excepciones, pueden clasificarse en diferentes tipos. Es importante entender los diferentes tipos de excepciones para manejarlas correctamente y mantener la estabilidad del programa.

Tipos de Excepciones

En programación, las excepciones se dividen principalmente en dos tipos:

1. Excepciones Verificadas (Checked Exceptions)

Las excepciones verificadas son aquellas que el compilador obliga a manejar. Es decir, el programador debe asegurarse de que estas excepciones se manejen adecuadamente, ya sea mediante un bloque `try-catch` o propagándolas hacia el llamador utilizando la palabra clave `throws`.

Las excepciones verificadas suelen ser causadas por condiciones externas al programa, como la falta de acceso a un archivo, una conexión de red perdida, o problemas al interactuar con la base de datos. Ejemplos comunes incluyen:

  • IOException: Ocurre cuando se producen errores al trabajar con archivos o flujos de entrada/salida.
  • SQLException: Se lanza cuando se produce un error al interactuar con una base de datos.
  • ClassNotFoundException: Se produce cuando el programa no puede encontrar una clase que se requiere durante la ejecución.

2. Excepciones No Verificadas (Unchecked Exceptions)

Las excepciones no verificadas son aquellas que no requieren que el compilador las controle. Estas excepciones suelen estar relacionadas con errores lógicos o del programa, como intentar dividir por cero, acceder a un índice fuera de los límites de un arreglo, o usar un objeto nulo.

Las excepciones no verificadas son subtipos de `RuntimeException` y se pueden evitar mediante una programación más cuidadosa. Ejemplos comunes incluyen:

  • ArithmeticException: Ocurre cuando se realiza una operación aritmética no válida, como dividir por cero.
  • NullPointerException: Se lanza cuando el código intenta acceder o modificar un objeto a través de una referencia nula.
  • ArrayIndexOutOfBoundsException: Se produce cuando se intenta acceder a un índice fuera de los límites de un arreglo.

3. Errores (Errors)

Los errores son condiciones que ocurren debido a fallos graves en el sistema, como problemas de memoria o un fallo en la JVM (Java Virtual Machine). Estos no son excepciones que deban manejarse en el código, ya que generalmente son irreparables y el programa debería finalizar. Los errores se heredan de la clase `Error` en Java.

Ejemplos de errores incluyen:

  • OutOfMemoryError: Indica que la máquina virtual de Java se ha quedado sin memoria.
  • StackOverflowError: Ocurre cuando se desborda la pila de llamadas, normalmente debido a una recursión infinita.

¿Cómo manejar las excepciones?

El manejo adecuado de excepciones es clave para asegurar la robustez de un programa. Dependiendo del tipo de excepción, se puede optar por gestionarlas de distintas formas, como:

  • Bloques try-catch: Usados para capturar excepciones y manejarlas de forma específica.
  • Palabra clave throws: Para propagar la excepción hacia el llamador si no se puede manejar en el mismo nivel.
  • Bloque finally: Usado para ejecutar código de limpieza, como liberar recursos, independientemente de si se produce una excepción o no.
