Herencia en Programación Orientada a Objetos

La herencia es un concepto fundamental en la programación orientada a objetos (OOP) que permite crear una nueva clase a partir de una clase existente, aprovechando y extendiendo sus características.

¿Qué es la Herencia?

La herencia permite que una clase adquiera propiedades y comportamientos de otra clase. Esto ayuda a reducir la duplicación de código y promueve la reutilización, haciendo que el código sea más eficiente y fácil de mantener.

En la herencia, la clase existente se denomina clase base (o clase padre), mientras que la nueva clase se denomina clase derivada (o clase hija).

Clase Base

Una clase base es una clase que se utiliza como modelo para crear otras clases. Contiene atributos y métodos que pueden ser heredados por otras clases. La clase base representa el comportamiento y las propiedades comunes que serán compartidos por las clases derivadas.

Por ejemplo, en una aplicación que maneja diferentes tipos de empleados, podríamos tener una clase base llamada Empleado, que contiene atributos y métodos comunes a todos los empleados, como el nombre, el ID y la función de calcular el salario.

Clase Derivada

Una clase derivada es una clase que hereda de la clase base. Puede utilizar los atributos y métodos de la clase base y también definir sus propias propiedades y comportamientos específicos. La clase derivada puede sobrescribir (modificar) métodos de la clase base si es necesario.

Siguiendo con el ejemplo anterior, podríamos tener clases derivadas como EmpleadoTiempoCompleto y EmpleadoPorHora, que heredan los atributos y métodos de la clase Empleado, pero agregan sus propias características, como un salario específico o una tarifa por hora.

Ejemplo de Herencia en C++

El siguiente código muestra cómo se puede implementar la herencia en C++ usando una clase base y una clase derivada:


#include <iostream>
using namespace std;

// Clase base
class Empleado {
public:
    string nombre;
    int id;

    Empleado(string nombre, int id) : nombre(nombre), id(id) {}

    void mostrarInformacion() {
        cout << "Nombre: " << nombre << ", ID: " << id << endl;
    }
};

// Clase derivada
class EmpleadoTiempoCompleto : public Empleado {
public:
    double salarioAnual;

    EmpleadoTiempoCompleto(string nombre, int id, double salario) : Empleado(nombre, id), salarioAnual(salario) {}

    void mostrarSalario() {
        cout << "Salario Anual: " << salarioAnual << endl;
    }
};

int main() {
    EmpleadoTiempoCompleto empleado1("Juan Perez", 101, 50000);
    empleado1.mostrarInformacion();
    empleado1.mostrarSalario();
    return 0;
}

En este ejemplo, la clase Empleado es la clase base y EmpleadoTiempoCompleto es la clase derivada que hereda de Empleado. La clase derivada puede acceder a los atributos y métodos de la clase base, además de definir su propio comportamiento.

Ventajas de la Herencia

  • Reutilización de código: Los atributos y métodos de la clase base se pueden reutilizar en las clases derivadas, evitando la duplicación de código.
  • Mantenimiento: Los cambios en la clase base se aplican automáticamente a las clases derivadas, lo que facilita el mantenimiento y actualización del código.
  • Jerarquía de clases: La herencia permite crear jerarquías de clases que reflejan la estructura del sistema o modelo que se está representando.

Resumen

La herencia es un mecanismo que permite crear nuevas clases basadas en clases existentes, promoviendo la reutilización y organización de código. La clase base proporciona el comportamiento y las propiedades comunes, mientras que las clases derivadas añaden o especializan sus propias características.
