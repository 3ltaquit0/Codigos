Clasificación de Archivos: Archivos de Texto y Binarios

Archivos de Texto

Los archivos de texto son archivos que contienen solo caracteres legibles por humanos. Estos archivos almacenan información en formato de texto sin ningún tipo de formato especial, aunque algunos pueden contener caracteres especiales como saltos de línea o tabulaciones.

Los archivos de texto pueden ser abiertos y editados fácilmente con cualquier editor de texto, como el Bloc de notas en Windows o el editor Vim en Linux. Los formatos más comunes de archivos de texto son:

  • Archivos .txt: El formato más básico de archivo de texto.
  • Archivos .csv: Archivos de valores separados por comas, usados para almacenar datos tabulares.
  • Archivos .log: Archivos de texto que almacenan registros de actividad de un sistema.

En los archivos de texto, cada carácter ocupa un número específico de bytes (por ejemplo, un byte por carácter en el formato ASCII). Estos archivos pueden ser procesados sin tener que convertir los datos, ya que están en un formato directamente legible.

Archivos Binarios

Los archivos binarios contienen datos que no están destinados a ser legibles directamente por humanos. En lugar de caracteres de texto, estos archivos almacenan datos en formato binario, que es el lenguaje que entiende la computadora.

Los archivos binarios pueden contener cualquier tipo de datos: texto codificado, imágenes, videos, programas ejecutables, etc. A diferencia de los archivos de texto, los archivos binarios no pueden ser leídos con un editor de texto simple, ya que su contenido no tiene una representación legible.

Algunos ejemplos de archivos binarios incluyen:

  • Archivos .jpg: Archivos de imagen comprimida en formato JPEG.
  • Archivos .exe: Archivos ejecutables para sistemas operativos Windows.
  • Archivos .mp3: Archivos de audio en formato comprimido MP3.
  • Archivos .pdf: Documentos en formato Portable Document Format.

Los archivos binarios requieren una interpretación especial para ser procesados correctamente, y se leen mediante flujos binarios en lugar de flujos de texto.

Comparación entre Archivos de Texto y Archivos Binarios

Características Archivos de Texto Archivos Binarios
Contenido Caracteres legibles por humanos Datos no legibles directamente
Formato Texto plano Datos en formato binario
Edición Se puede editar con cualquier editor de texto Requiere programas específicos para abrir y editar
Uso común Documentos, registros, código fuente Imágenes, audio, video, programas ejecutables

Operaciones con Archivos de Texto y Binarios

La principal diferencia entre los archivos de texto y los binarios es cómo se leen y escriben. Los archivos de texto pueden ser manejados fácilmente con flujos de caracteres, mientras que los archivos binarios requieren flujos binarios para ser procesados correctamente.

A continuación, un ejemplo de cómo abrir, leer y escribir en un archivo binario en Java: 


import java.io.*;

public class EjemploArchivoBinario {
    public static void main(String[] args) {
        // Escritura en archivo binario
        try (FileOutputStream archivoSalida = new FileOutputStream("ejemplo.bin")) {
            byte[] datos = {1, 2, 3, 4, 5};
            archivoSalida.write(datos);
        } catch (IOException e) {
            e.printStackTrace();
        }

        // Lectura de archivo binario
        try (FileInputStream archivoEntrada = new FileInputStream("ejemplo.bin")) {
            int contenido;
            while ((contenido = archivoEntrada.read()) != -1) {
                System.out.print(contenido + " ");
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
