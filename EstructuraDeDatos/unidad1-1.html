Clasificación de las Estructuras de Datos

¿Alguna vez te has preguntado cómo las computadoras organizan la información para que todo funcione rápido y sin problemas? Bueno, ahí es donde entran las **estructuras de datos**. Básicamente, son formas de guardar y manipular datos para que los programas trabajen de manera eficiente.

1. Estructuras de Datos Simples

Estas son las más básicas, las que manejamos en cualquier lenguaje de programación. Aquí cada variable guarda un solo valor a la vez. Algunos ejemplos:

  • Enteros: Números sin decimales, como `int` o `long`.
  • Flotantes: Números con decimales, como `float` y `double`.
  • Cadenas de texto: Para palabras o frases, usamos `String`.
  • Booleanos: Solo pueden ser `true` o `false` (como un interruptor: encendido o apagado).

2. Estructuras de Datos Compuestas

Estas nos permiten guardar varios valores en una sola variable, en lugar de crear una por cada dato. Imagínatelas como una caja que puede almacenar muchas cosas adentro.

  • Arreglos: Son listas de tamaño fijo donde cada elemento tiene una posición (índice).
  • Listas: Parecidas a los arreglos, pero su tamaño puede crecer o reducirse dinámicamente.
  • Mapas (Diccionarios): Guardan datos en pares **clave-valor**, como una agenda de contactos.

Ejemplo en Java:


// Arreglo en Java
int[] numeros = {1, 2, 3, 4};

// Lista en Java
import java.util.ArrayList;
ArrayList lista = new ArrayList<>();
lista.add("Juan");
lista.add("María");

// Mapa (Diccionario) en Java
import java.util.HashMap;
HashMap edades = new HashMap<>();
edades.put("Juan", 25);
edades.put("María", 30);
System.out.println(edades.get("Juan")); // Muestra la edad de Juan

3. Estructuras de Datos Lineales

Aquí los datos se organizan uno detrás del otro, como una fila de personas esperando su turno.

  • Pilas (Stack): Funciona como una torre de platos, el último en entrar es el primero en salir (**LIFO: Last In, First Out**).
  • Colas (Queue): Funciona como una fila en el banco, el primero en entrar es el primero en salir (**FIFO: First In, First Out**).

Ejemplo de una Pila en Java:


import java.util.Stack;

public class EjemploPila {
    public static void main(String[] args) {
        Stack pila = new Stack<>();
        pila.push(1);  // Se apila (agrega)
        pila.push(2);
        pila.push(3);
        System.out.println(pila.pop()); // Sale el último: 3
    }
}

Ejemplo de una Cola en Java:


import java.util.LinkedList;
import java.util.Queue;

public class EjemploCola {
    public static void main(String[] args) {
        Queue cola = new LinkedList<>();
        cola.add("A");
        cola.add("B");
        cola.add("C");
        System.out.println(cola.poll()); // Sale el primero: "A"
    }
}

4. Estructuras de Datos No Lineales

Aquí los datos no se guardan en una fila, sino que se conectan de forma más compleja. Son muy usados en bases de datos y sistemas de navegación.

  • Árboles: Tienen una estructura jerárquica con un **nodo raíz** y varios **nodos hijos**.
  • Grafos: Son conjuntos de nodos conectados por enlaces (como un mapa de carreteras o redes sociales).

Ejemplo de Árbol Binario en Java:


class Nodo {
    int valor;
    Nodo izquierda, derecha;

    public Nodo(int valor) {
        this.valor = valor;
        izquierda = derecha = null;
    }
}

// Clase principal que crea un árbol binario simple
public class Arbol {
    Nodo raiz;

    public Arbol(int valor) {
        raiz = new Nodo(valor);
    }

    public static void main(String[] args) {
        Arbol arbol = new Arbol(10);
        arbol.raiz.izquierda = new Nodo(5);
        arbol.raiz.derecha = new Nodo(15);
        System.out.println("Árbol creado con raíz en: " + arbol.raiz.valor);
    }
}

Conclusión

Como ves, las estructuras de datos son la base de cualquier programa. Dependiendo del problema que estés resolviendo, necesitarás usar una u otra.

- Si necesitas **orden y velocidad**, usa **listas o arreglos**.

- Si el orden de entrada y salida es importante, usa **pilas o colas**.

- Si necesitas organizar datos jerárquicamente, usa **árboles**.

- Si trabajas con relaciones entre elementos, usa **grafos**.
